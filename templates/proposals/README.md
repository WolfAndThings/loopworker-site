# Proposal HTML Templates

Use this folder as the default starting point for future client-facing proposal pages.

## Note

The Vidyfy proposal pages established a strong visual direction:

- dark editorial layout
- high-contrast typography
- clean card system
- good spacing on desktop and mobile
- easy to share as a standalone HTML page

This should be the baseline style for future proposal pages unless there is a clear reason to change it.

## Included Templates

- `recommended-game-plan-template.html`
  - best when you want to lead with a single recommendation and staged plan
- `options-comparison-template.html`
  - best when you want to show 2-3 scoped options side by side

## Recommended Workflow

1. Copy the template into `proposals/<client-slug>/index.html` or a similar path.
2. Replace the `{{PLACEHOLDER}}` tokens.
3. Keep `noindex, nofollow` unless the page is meant to be public.
4. Prefer direct language to the client: `you`, `your team`, and the company name.
5. Keep deliverables tight enough that the page does not imply a bigger scope than the retainer supports.

## Design Reminder

The page should feel:

- polished
- premium
- easy to skim
- convincing without sounding salesy

If a future proposal starts looking generic, use the Vidyfy pages as the reference:

- `/proposals/vidyfy/`
- `/proposals/vidyfy/options.html`
